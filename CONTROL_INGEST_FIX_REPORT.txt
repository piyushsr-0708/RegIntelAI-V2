ROOT CAUSE
----------

File: backend/database/services/pipeline_ingestion_service.py
Method: ingest_document() - Line 82

ISSUE:
The ingestion service was IGNORING the control_id field from the JSON and 
generating hashed IDs instead:

    # OLD CODE (BROKEN)
    deterministic_ctrl_id = self._generate_control_hash(ctrl_data)

The _generate_control_hash() method creates hash-based IDs like "CTRL_abc123..." 
by hashing control_name, implementation_category, control_type, and 
implementation_method fields.

PROBLEMS WITH HASHING APPROACH:
1. Loses document traceability (no document ID prefix)
2. Control Deriver generates proper IDs (e.g., MD10190_ctrl_req5_1) that get discarded
3. Hash collisions when fields are empty or similar
4. Frontend queries by document ID prefix fail (returns 0 controls)
5. Corrupted control names when hashing empty/malformed fields

THE FIX
-------

Changed Line 82-83 to preserve the control_id from JSON:

    # NEW CODE (FIXED)
    # Use the control_id from JSON if available, otherwise fall back to hash
    deterministic_ctrl_id = ctrl_data.get("control_id") or self._generate_control_hash(ctrl_data)

This preserves the structured control IDs generated by Control Deriver:
- Format: {document_id}_ctrl_{requirement_id}_{sequence}
- Example: UP20260715_0001_ctrl_req5_1
- Enables document-scoped queries
- Maintains full provenance chain
